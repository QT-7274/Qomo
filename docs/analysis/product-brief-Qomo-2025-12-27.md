# Product Brief: Qomo

**Date:** 2025-12-27
**Author:** drogbaqu

---

## Executive Summary

Qomo 是一个面向高频使用对话式 AI 的用户的可视化 Prompt 资产库与组装工具。它将 Prompt 从“每次重写的文本”升级为“可复用的资产”：用户可以把常用 Prompt 拆分为模块（如角色、目标、约束、输出格式、检查清单等），也可以维护为带变量的模板。在每次提问前，用户通过“模块化拼装 + 变量填充 + 约束包选择”，生成一份结构清晰、约束明确、可直接使用的最终 Prompt，并一键复制/导出给用户选择的外部 AI（一期不做模型直连）。未来，Qomo 支持导入/导出与社区分享，形成可迁移、可协作迭代的 Prompt 资产库。

---

## Core Vision

### Problem Statement

对话式 AI 的提问质量高度依赖 Prompt 的结构与约束，但高频用户在每次提问前都要重复编写或临时拼凑 Prompt（角色、目标、边界、输出格式、背景信息等）。这个过程繁琐、容易遗漏关键约束，导致提问准备成本高、输出质量不稳定、优质 Prompt 难以沉淀复用与迭代。

### Primary Users & Core Use Cases（核心用户与场景）

- 希望提升工作效率的工作者：日常要写方案、总结、邮件、汇报、PRD、脚本说明等，反复向对话式 AI 提问，但每次从零组织 Prompt 很耗时。
- 重度与 AI 交互的使用者：需要在重复性工作场景/工作任务中持续复用 Prompt，追求“少返工、输出更稳定”。

### User Segments（用户分层）

- 新手用户：更倾向“单次 Prompt 直接用”，需要最短路径（选模板 → 填 1-2 个变量 → 一键导出），以及导出前的轻量提醒（占位符未填/输出格式提示）。
- 高阶 Prompt 工程用户：更倾向“多次复用 + 组装”，需要模块化资产、变量体系、组合预览与可维护性（如标签/版本/导入导出）。

### Primary vs Secondary Users（主用户与次用户）

- 主用户：一期主要聚焦上述两类（新手与高阶）的个人使用者。
- 次用户：一期暂不区分（团队协作/社区分享者作为后续扩展方向）。

### Problem Impact

- 重复劳动：相似问题反复从零组织 Prompt，提问前成本高。
- 质量波动：约束/格式/边界遗漏 → 多轮纠错与返工。
- 资产不可管理：优质 Prompt 分散在聊天记录、笔记、剪贴板，缺乏结构与版本。
- 难以迁移协作：个人模板难导出；团队最佳实践难共享、难迭代。

### Why Existing Solutions Fall Short

- 通用笔记/文档/剪贴板：能存放但难以“结构化资产化”，缺少模块化复用、变量填充、组合预览与导出前校验。
- Prompt 展示/检索型库：偏“发现与收藏”，但不覆盖“按我的工作流组装生成一份可直接使用的最终 Prompt”。
- 纯文本模板：维护成本高、结构不可见、组合与版本演进困难；很难把“有效约束”沉淀为可复用组件。

### Proposed Solution

Qomo 提供一个以“Prompt 资产化 + 生成工作流”为核心的工作台：

1) Prompt 资产模型（可视化管理）
- 模块（Module）：可复用片段（如 Role / Goals / Constraints / Output Format / Examples / Safety），支持标签、搜索、复用。
- 模板（Template）：由多个模块组成，并包含变量槽位（Variables），用于快速定制与复用。
- 约束包（Constraint Packs）：将常见有效约束产品化为可组合选项，按场景选择：
  - 输出约束：要求结构/格式（如 Markdown 小标题、表格、JSON Schema）。
  - 边界约束：定义范围与禁止项（如“不确定先提问”“不要编造”“只使用给定上下文”）。
  - 质量约束：引导更可靠流程（如“先列假设与缺失信息”“输出前自检清单”）。

2) 生成工作流（面向提问前一公里）
- 选择模板/模块 → 填变量（本次问题/上下文/偏好）→ 勾选约束包 → 生成最终 Prompt（结构化预览）。
- 一期交付：复制/导出文本（不直连模型，平台无关）。

3) 轻量校验（Preview/Lightweight Check）
- 导出前做最小但高价值的检查：占位符未填、关键约束缺失提示、疑似敏感信息提示/遮罩建议、输出结构约束提醒（如需 JSON/表格时）。

4) 可迁移与可演进
- 导入/导出模板与模块（便于迁移、备份、团队共用）。
- 后续：社区分享与迭代（评分/版本/衍生），形成“可组合的 Prompt 资产网络”。

### Safety & Trust Boundaries（安全与信任边界）

由于 Prompt 可能包含敏感信息（个人隐私、公司机密、账号密钥等），Qomo 将把“安全与可控”作为核心体验的一部分：

- 一期不直连模型：只生成并导出文本，降低数据外流与平台锁定风险。
- 资产与单次问题分离：模板/模块作为可维护资产；单次问题默认不沉淀或可控沉淀，降低敏感内容长期存储风险。
- 非阻断式安全提示：导出前轻提示敏感信息并提供一键遮罩，同时允许用户继续导出。
- 导入可信机制（面向未来社区）：展示来源/版本/变更，并对潜在“隐藏指令/高风险语句”进行提示，避免资产被污染。

### Key Differentiators

- Qomo 同时提供 Prompt 资产库（收藏/管理）与生成工作流（组装→预览/轻量校验→导出），让 Prompt 成为可控、可复用、可迁移的生产资料。
- 约束包机制：把“有效约束”产品化为可选组件，按场景选择与复用，降低学习与试错成本。
- 轻主路径优先：为新手与高频提问提供“选模板→填变量→一键生成并复制”的最短路径，高级选项折叠；同时为高阶用户提供模块化拼装能力。
- 一期清晰边界：只做复制/导出，把生成工作流与资产模型先打扎实；未来再扩展社区与生态。

## Success Metrics

### North Star Metric（北极星指标）

- **WAEP（Weekly Active Exporters with Reuse）**：每周至少有一次“生成并导出/复制”，且当次会话包含“复用”（模块/模板/历史组合任一）并**实际参与生成**。
  - 目的：把“产出价值 + 复用减少返工”绑定在一起，避免只追 DAU/导出次数等虚荣指标。

### Activation（两段式激活）

- **Activation-1（首次价值）**：首次会话完成 `生成 → 导出/复制`。
- **Activation-2（复用价值）**：后续会话完成 `复用（模块/模板/历史组合并实际参与生成）→ 生成 → 导出/复制`。

### User Success Metrics（用户成功）

- **更快产出高质量 Prompt**
  - 指标：核心路径完成率（进入 → 生成 → 导出/复制）、完成用时分布（如 p50/p95）。
- **复用资产减少返工**
  - 指标：复用会话占比、复用次数/人、复用资产覆盖（模块/模板/历史组合）。
- **Aha 时刻达成（随取随用、随时组装）**
  - 指标：首次达成 Aha 行为占比（示例：创建≥1模块 + 生成≥1次 + 导出≥1次）。

### Business Objectives（业务目标）

- **拉新**：新用户增长、渠道转化（如分享/邀请带来的新增）。
- **留存**：次日/7日/30日留存（按“有效活跃”口径）。
- **活跃**：DAU/WAU/MAU、人均生成/导出/复用次数。
- **口碑传播**：分享/邀请行为与回流（优先用可归因链路的数据）。

### Key Performance Indicators（关键 KPI）

- **核心漏斗**：进入工作台 → 选择模板/组合模块 → 生成 → 导出/复制 转化率。
- **复用效率**：复用会话占比、复用资产占比（只统计“实际参与生成”的复用）。
- **产出强度**：人均导出/复制次数、人均生成次数。
- **粘性与留存**：DAU/WAU、次日/7日/30日留存。

### Quality Gates（质量门槛：前端 + 云函数/边缘函数 + KV）

> 质量门槛优先：当核心链路失败率或 p95/p99 延迟显著恶化时，增长类指标不作为成功判断依据。

- **核心链路失败率**：生成/导出/复制的 `success=false` 占比（按版本/环境/地区拆分）。
- **函数调用可用性**：错误率（超时/限流/5xx）+ p95/p99 延迟。
- **KV 读写可用性**：读写错误率 + 重试/冲突（如存在）。
- **前端稳定性与性能**：JS 错误/白屏、关键交互耗时（如“点击生成→预览可用”的 p95）。

### Valid Export（有效导出，N = 5 分钟）

- **定义**：导出后 5 分钟内是否出现频繁“再次生成/再次导出/反复编辑”。
- **目的**：作为“少返工”的代理指标，衡量导出是否足够可用。

## MVP Scope

### Core Features

- **价值主线（复用优先）**
  - 把 Prompt 变成可复用资产：随取随用、随时组装、一键导出
  - 核心目标：通过模块/模板/历史组合的复用，减少重复劳动与返工

- **Prompt 资产化管理（必须）**
  - 模块（Module）：创建/编辑/删除、标签、搜索、复用
  - 模板（Template）：由多个模块组成，支持变量槽位，用于快速定制与复用
  - 约束包（Constraint Packs）：MVP 仅提供少量内置约束包（克制范围，避免膨胀为第二套模板系统）

- **生成工作流（必须）**
  - 选择模板/模块 → 填变量 → 选择约束包 → 生成最终 Prompt（结构化预览）
  - 支持“复用上次组合/最近组合”（快速进入状态）

- **导出与使用（必须）**
  - 一键复制/导出文本（一期不做模型直连）

- **导出前轻量校验（G，MVP 必做）**
  - 占位符未填提醒（高亮变量 + 导出前提示，默认非阻断）
  - 输出格式约束提醒（如选择 JSON/表格等时给出一致性提示，非阻断）
  - 空内容/全占位符阻断导出（唯一硬拦截）
  - 敏感信息提示/遮罩建议（非阻断）
  - 原则：除“空内容/全占位符”外，其余校验均为非阻断提示

- **新手也能达成 Aha（必须）**
  - 新手首次体验目标路径：创建 1 个模块 → 用模板生成 → 复制导出

> 说明：由于 Top 1 具体业务场景尚未选定，MVP 先交付“跨场景通用链路”，
通过模板/模块/复用来覆盖写方案、总结、邮件、汇报、PRD、脚本说明等多类输出。

### Out of Scope for MVP

- 社区/市场能力：分享、评分、衍生（不做）
- 团队协作与权限系统（不做）
- 复杂版本管理：分支、对比、回滚等（不做）

### MVP Success Criteria

- **核心判断：用户使用率/使用频次提升**
  - 优先关注活跃（DAU/WAU）、留存（次日/7日/30日）
- **价值兑现：从首次价值到复用价值**
  - Activation-1（首次生成→导出/复制）稳定达成
  - Activation-2（复用参与生成→导出/复制）出现明显增长趋势
- **质量门槛稳定（前端 + 云函数/边缘函数 + KV）**
  - 核心链路失败率与 p95/p99 延迟保持可控

### Future Vision

- **团队共享 Prompt 资产网络**
  - 团队沉淀模块、模板、约束包，形成可治理、可迁移的知识资产
- **市场/社区生态**
  - 模板与模块的发布、评分、衍生与版本演进，形成“可组合的 Prompt 资产网络”
